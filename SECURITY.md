# Security Policy

## Private disclosure

Please do not report security issues publicly.

If you discover a vulnerability, exposed secret, leaked license material, or a way to bypass protected release assets, contact the maintainer privately.

## What is intentionally not public

This repository does not publish:

- generated license keys;
- access key vaults;
- private build scripts for sealing keys;
- release installers and local build outputs;
- debug, audit, probe, and temporary files.

If any of those files appear in a public branch or release by mistake, consider it a security incident.
